Core Types

IgDiscover.Config — Type
Config

Complete IgDiscover configuration. Loaded from a TOML file merged with defaults.

Note: merge_program, flash_maximum_overlap, minimum_merged_read_length, and stranded are parsed and stored but not yet used (reserved for future read-merge and strand-aware support).

source
IgDiscover.FilterStats — Type
FilterStats

Counts from preprocessing filter: total, has_vj_assignment, has_no_stop, good_v_evalue, good_v_coverage, good_j_coverage, has_cdr3. Supports + for summing stats.

IgDiscover.read_fasta — Function
read_fasta(path; limit=0) -> Vector{FastaRecord}

Read records from a FASTA or FASTQ file (plain or gzipped). Format is auto-detected by the first character ('>' = FASTA, '@' = FASTQ). Sequences are uppercased. If limit > 0, stop after that many records. Dispatch is via SequenceFormat.

IgDiscover.sanitize_imgt_sequence — Function
sanitize_imgt_sequence(seq) -> String

Remove IMGT gap characters (dots) from a nucleotide sequence and uppercase. IMGT databases use '.' to represent gaps in their numbering system; these must be removed before use with IgBLAST or alignment tools.

source
IgDiscover.write_sanitized_imgt — Function
write_sanitized_imgt(path_in, path_out)

Read IMGT FASTA from pathin and write to pathout with:

  • Headers replaced by allele names (e.g. IGHV1-1801, IGHD3-301, IGHJ1*01)
  • IMGT gap dots removed from sequences

This ensures IgBLAST and downstream tools see clean identifiers and ungapped sequences.

IgDiscover.deduplicate_by_consensus — Function
deduplicate_by_consensus(filtered) -> DataFrame

Keep one row per unique consensus sequence (same as Python igdiscover). Julia discovery emits multiple rows per gene (one per error-rate window or cluster); Python emits one per gene. Collapse duplicates by keeping the row with largest cluster_size per consensus, then first by name for tie-break.

IgDiscover.rename_genes — Function
rename_genes(discovered_fasta, database_fasta, output_fasta)

Rename discovered gene sequences based on closest match to original database. Each discovered sequence gets the name of its closest database gene, with allele suffixes (*01, *02, ...) assigned by cluster order.
